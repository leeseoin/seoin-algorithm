# 코딩테스트 학습 저장소

이서인의 코딩테스트 학습/풀이 저장소. 프로그래머스 위주, 파이썬 사용. 이 문서는 **전체 디렉토리 구조와 각 문서의 역할** 을 정리한 인덱스입니다.

## 문서 역할별 안내

### 📌 메타 / 상시 참조

| 문서 | 역할 | 언제 보나 |
|---|---|---|
| `CLAUDE.md` | 전체 인덱스 (이 파일) | 어디 있는지 기억 안 날 때 |
| `코테_공부_가이드.md` | 학습 방향, 사고 순서, 체크리스트 | 막막할 때, 학습 페이스 점검 |
| `파이썬_문법_레퍼런스.md` | 내장함수/도구 사전 (15개 카테고리) | 풀다가 "이거 어떻게 쓰더라" |

### 🎯 학습 (개념 정리)

| 폴더 | 내용 |
|---|---|
| `학습/자료구조/` | 스택, 큐, 링크드리스트, 해시테이블, 그래프, 트리 (6개) |
| `학습/알고리즘/` | 시뮬·완탐·BFS/DFS (1순위), DP·다익스트라·이분탐색·위상정렬 (2순위) |
| `학습/SQL/` | 기본문법, 조인, 집계, 서브쿼리, 윈도우 함수 (5개) |
| `학습/학습계획.md` | 우선순위, 의존성 그래프, 4주 로드맵 |

각 문서는 **10~15분 분량** — 출퇴근에 한 편씩.

### 📝 풀이 / 회고

| 위치 | 내용 |
|---|---|
| `programmers/LV{N}/*.py` | 실제 풀이 코드 |
| `programmers/LV{N}/문제별_회고.md` | 문제별 이슈/해결/배운 점 누적 기록 |

새 문제 풀고 나서 막혔던 부분이 있으면 회고 추가. 1주/1달 후 다시 읽기.

### 🤖 스킬

| 스킬 | 동작 |
|---|---|
| `/cote-summary [문제파일.py]` | 대화 맥락에서 이슈 추출 → 해당 LV의 `문제별_회고.md`에 누적 추가 |

## 주요 의존성 / 학습 순서

```
[자료구조 기반]
  스택 ─┐
  큐 ───┼── BFS/DFS ─── 다익스트라
  해시 ┐│              (가중치 그래프)
  그래프┘│
  트리 ─┘
              완탐 ─── DP
              이분탐색 (독립)
              위상정렬 (그래프 기반)
```

자세한 의존성과 일주별 계획은 `학습/학습계획.md` 참고.

---

## 학습 사이클

```
1. 학습 문서 1편 정독       (10~15분, 출퇴근)
2. 관련 문제 1~2개 풀이     (30분~1시간, 집)
3. 막힌 부분/배운 점 기록    → 문제별_회고.md
4. 1주일 후 회고 다시 읽기
5. 1달 후 학습 문서 통째로 재독
```

## 모바일 학습 흐름

1. 이 저장소를 GitHub에 push
2. **GitHub 모바일 앱** 으로 출퇴근 중 마크다운 열람
3. 메모하고 싶은 게 생기면 GitHub 앱에서 직접 편집/커밋
4. 집에 와서 코드로 옮겨보기

## 자주 보는 경로 빠르게

- "정렬은 어떻게 하더라" → `파이썬_문법_레퍼런스.md` § 1 기본 집계
- "이 문제 시간 초과 나는데" → `코테_공부_가이드.md` § 8 막힐 때 대처법
- "스택 어디 쓰지" → `학습/자료구조/1_스택.md`
- "오늘 뭐 공부하지" → `학습/학습계획.md` 4주 로드맵
- "비슷한 문제 전에 어떻게 풀었더라" → `programmers/LV{N}/문제별_회고.md`

## 문서 업데이트 정책

- **`programmers/LV{N}/문제별_회고.md`**: `/cote-summary` 스킬이 자동 추가
- **공용 문서 (가이드/레퍼런스)**: 새로운 함정/도구 발견 시에만 사용자 확인 후 갱신
- **학습 문서 (`학습/`)**: 사용자가 추가하거나 본인이 학습 후 메모 추가
- **CLAUDE.md (이 파일)**: 디렉토리 구조 변경 시 갱신
