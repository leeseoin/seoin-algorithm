## 1. 두수나눗셈.py — 정수 부분 추출

**문제 요약**
`num1 / num2 * 1000`의 **정수 부분**(소수점 이하 버림)을 반환.

**처음 시도한 코드**
```python
def solution(num1, num2):
    answer = (num1 / num2) * 1000
    format_answer = round(answer, 0)
    return format_answer
```

**발생한 이슈**
샘플 4개는 통과했지만 채점 시 18개 중 **6개 실패 (66.7점)**.

**원인**
- 문제는 "**정수 부분**"(버림)을 요구했는데 `round()`는 **반올림**.
- 추가로 파이썬 `round()`는 일반 반올림이 아니라 **banker's rounding** ("5는 짝수 쪽으로").
  - 예: `round(0.5) = 0`, `round(2.5) = 2`
- 샘플은 `.5` 미만/초과 케이스가 많아 우연히 다 통과한 것.
- 실패 예: `2/3 * 1000 = 666.666...` → 정답 666, `round` 결과 667.

**해결 방법**
```python
def solution(num1, num2):
    return int(num1 / num2 * 1000)

# 또는 정수 나눗셈
def solution(num1, num2):
    return num1 * 1000 // num2
```

**핵심 문법 / 배운 점**
- "정수 부분", "버림", "내림" → **`int()` 또는 `//`** (`round()` 금지)
- `round()` / `f"{x:.0f}"` 는 **반올림**이고, 후자는 **str 반환**까지 됨
- 비교표

  | 방식 | 동작 | 반환 타입 |
  |---|---|---|
  | `int(x)` | 소수점 버림 | int |
  | `a // b` | 정수 나눗셈 (버림) | int/float |
  | `round(x, 0)` | 반올림 (banker's) | float |
  | `f"{x:.0f}"` | 반올림 + 문자열 | str |

## 2. 배열뒤집기.py — 리스트 뒤집기와 in-place 메서드

**문제 요약**
정수 리스트 `num_list`를 뒤집어서 반환.

**처음 시도한 코드**
```python
def solution(num_list):
    reverse_num_list = num_list[::-1]
    reverse_num_list2 = num_list.reverse()
    reverse_num_list3 = list(reversed(num_list))
    return reverse_num_list2   # ← None이 반환됨
```

**발생한 이슈**
- `reverse_num_list2`가 `None`으로 출력됨
- `.reverse()`를 중간에 호출해서 **원본이 뒤집혀버림** → 뒤이은 `reversed(num_list)`가 이중 반전되어 결과가 원래 순서로 나오는 부작용

**원인**
- `list.reverse()`는 **in-place 메서드** — 원본을 직접 바꾸고 **`None`을 반환**.
- 파이썬에서 원본을 바꾸는 메서드는 거의 다 `None`을 반환함 (`.sort()`, `.append()`, `.extend()`, `.clear()` ...).
- "방 정리해줘"라는 **명령**과 같음 — 명령 자체는 빈손으로 돌아오지만 방(원본) 상태는 바뀌어 있음.

**해결 방법**
```python
# 방법 A — in-place 후 객체 자체를 리턴
def solution(num_list):
    num_list.reverse()
    return num_list

# 방법 B — 새 리스트 생성 (가장 자주 씀)
def solution(num_list):
    return num_list[::-1]

# 방법 C — reversed() + list()
def solution(num_list):
    return list(reversed(num_list))
```

**핵심 문법 / 배운 점**
- in-place 메서드는 **`None` 반환**, 절대 `x = arr.reverse()` 같이 받지 말 것
- "동사형 메서드" vs "**-ed 형 함수**"의 짝
  - `arr.sort()` ↔ `sorted(arr)`
  - `arr.reverse()` ↔ `reversed(arr)`, `arr[::-1]`
- 슬라이싱 `[::-1]` 은 **새 리스트** 반환 (원본 그대로)
- 문자열은 불변이라 `.reverse()` 메서드가 **없음**, 슬라이싱이나 `reversed()`만 가능

## 3. 뒤집한문자열.py — 문자열 누적과 슬라이싱

**문제 요약**
문자열 `my_string`을 거꾸로 뒤집어 반환.

**처음 시도한 코드**
```python
def solution(my_string):
    reversed_string = ""
    answer = ""
    for i in my_string:
        answer = reversed_string + i   # reversed_string은 계속 ""
    return answer
```

**발생한 이슈**
입력 `"jaron"` 에 대해 결과가 `"n"` (마지막 글자 한 개만 나옴).

**원인**
1. **누적이 안 됨**: `reversed_string`을 초기화 후 **한 번도 갱신하지 않아** 매 루프마다 `answer = "" + i` → 마지막 글자만 남음.
2. **뒤집기 방향**: 뒤집으려면 **새 글자를 앞에 붙여야** 하는데 뒤에 붙이고 있었음.

**해결 방법**
```python
# 반복문 누적 패턴 (원리 이해용)
def solution(my_string):
    answer = ""
    for ch in my_string:
        answer = ch + answer   # 새 글자를 "앞에" 붙임
    return answer

# 슬라이싱 (실전 추천)
def solution(my_string):
    return my_string[::-1]
```

**핵심 문법 / 배운 점**
- **문자열은 불변(immutable)** — `+=`은 새 문자열을 만들어 재할당하는 것
- `+=`로 누적하면 매 루프마다 전체 복사 → **O(n²)** (시간 초과의 주범)
- 길이 1,000 → 약 50만 번 복사 / 길이 10,000 → 5천만 번
- 누적이 필요하면 **`list.append()` 후 `"".join(list)` 패턴** (O(n))
  ```python
  parts = []
  for ch in s:
      parts.append(ch)
  answer = "".join(parts)
  ```
- **슬라이싱이 빠른 이유**:
  1. C로 구현되어 있음 (파이썬 인터프리터 거치지 않음)
  2. 한 번에 새 문자열을 만들어 O(n)

## 4. 짝수의합.py — iterable과 range()

**문제 요약**
`0`부터 `n`까지 (포함) 짝수의 합을 반환.

**처음 시도한 코드**
```python
def solution(n):
    answer = 0
    for i in n+1:           # ← 여기서 에러
        if i % 2 == 0:
            answer += i
    return answer
solution(10)
print(solution)             # 함수 객체 자체가 출력됨
```

**발생한 이슈**
```
TypeError: 'int' object is not iterable
```

**원인**
- `for x in ???:` 의 `???` 자리에는 **iterable**(반복 가능한 객체)만 올 수 있음.
- 리스트/문자열/튜플/딕셔너리/`range`는 iterable, **정수는 iterable이 아님**.
- `n+1`은 그냥 숫자 하나(예: `11`) — "11에서 하나씩 꺼내"라는 명령은 불가능.
- 별개로 `print(solution)`은 함수를 호출한 게 아니라 **함수 객체 자체**를 출력 → `<function solution at 0x...>` 형태로 나옴. 호출하려면 `()` 필요.

**해결 방법**
```python
def solution(n):
    answer = 0
    for i in range(n+1):    # 0, 1, 2, ..., n
        if i % 2 == 0:
            answer += i
    return answer

# 한 줄 풀이
def solution(n):
    return sum(range(0, n+1, 2))
```

**핵심 문법 / 배운 점**
- **iterable** = "하나씩 꺼낼 수 있는 객체" (리스트, 문자열, 튜플, dict, set, range, ...)
- 정수/실수/불리언은 iterable **아님** → `for i in 10:` 같이 못 씀
- **`range()`는 끝값 미포함**
  ```python
  range(11)        # 0, 1, ..., 10  (11 미포함!)
  range(1, 11)     # 1, 2, ..., 10
  range(0, 11, 2)  # 0, 2, 4, ..., 10
  ```
- "0부터 n까지 포함" 하고 싶으면 `range(n+1)`
- **함수 객체 vs 함수 호출**
  - `solution` → 함수 자체 (객체)
  - `solution(10)` → 함수를 실행한 결과

## 5. 순서쌍의개수.py — 시간복잡도와 약수 카운트

**문제 요약**
자연수 `n`이 주어질 때, `a * b == n` 인 자연수 순서쌍 `(a, b)`의 개수를 반환. (`1 ≤ n ≤ 1,000,000`)

**처음 시도한 코드**
```python
def solution(n):
    answer = 0
    for i in range(1, n+1):
        for a in range(1, n+1):
            if i * a == n:
                answer += 1
    return answer
```

**발생한 이슈**
**시간 초과**. 샘플(n=20, n=100)은 통과해도 큰 n에서 실패.

**원인**
- 이중 for문 → 총 **n × n = n²** 번 연산.
- `n = 10⁶`이면 **10¹²번 (1조 번)** — 파이썬은 1초에 약 10⁷~10⁸번 처리 → **수 시간** 걸림.
- 채점기 시간 제한 1~2초를 한참 초과.
- 핵심 인사이트를 놓침: `i * a == n` ⇔ **`i`가 `n`의 약수**. 즉 이 문제는 "n의 약수 개수 세기"로 환원됨.

**해결 방법**
```python
# O(n) — 한 번만 돌면 됨
def solution(n):
    answer = 0
    for i in range(1, n+1):
        if n % i == 0:
            answer += 1
    return answer

# O(n) 한 줄 풀이
def solution(n):
    return sum(1 for i in range(1, n+1) if n % i == 0)

# O(√n) — 약수는 쌍으로 등장하는 성질 활용
def solution(n):
    answer = 0
    i = 1
    while i * i <= n:
        if n % i == 0:
            answer += 2 if i * i != n else 1
        i += 1
    return answer
```

**핵심 문법 / 배운 점**
- **시간복잡도(Big-O)** = "입력 n이 커질 때 연산량이 어떻게 늘어나는가"
- 이중 for문이 보이면 **"진짜 두 번 다 돌 필요 있나?"** 의심
- `a * b = n` 형태 문제 → **약수 문제로 환원 가능**
- 약수 쌍의 성질: `i`가 약수면 `n/i`도 약수 → **√n까지만 돌아도 됨**
- n 크기별 허용 복잡도는 `코테_공부_가이드.md` 참고

## 6. 모음제거.py — 누적 패턴과 replace 함정

**문제 요약**
영어 문자열 `my_string`에서 모음 `a, e, i, o, u`를 모두 제거한 문자열 반환.

**처음 시도한 코드 (1차)**
```python
def solution(my_string):
    answer = ''
    m = ["a","e","i","o","u"]
    for i in m:
        my_sting = my_string.replace(i, "")   # ← 오타 + 결과를 안 씀
        print(my_string)
    return answer                              # ← 빈 문자열 그대로
```

**1차 이슈**: 결과가 항상 `""` (빈 문자열).

**1차 원인**:
1. **변수명 오타** `my_sting` (r 빠짐) → 새 변수가 만들어지고 원본은 안 바뀜. 파이썬은 변수 선언이 없어서 오타도 그냥 새 변수로 받아들임 → **NameError 안 남**, 발견 어려움.
2. `replace()` 결과를 `answer`에 저장하지 않고 다른 변수에 저장 → `answer`는 초기값 `''` 그대로 리턴.

**처음 시도한 코드 (2차 — 오타 고친 후)**
```python
def solution(my_string):
    answer = ''
    m = ["a","e","i","o","u"]
    for i in m:
        answer = my_string.replace(i, "")     # ← 매번 원본에서 시작
    return answer
```

**2차 이슈**:
- `"bus"` → `"bs"` ✅ (운 좋게 통과)
- `"nice to meet you"` → `"nice to meet yo"` ❌ (u만 지워짐)

**2차 원인** (진짜 핵심):
- **누적이 안 됨**. 매 루프마다 `my_string.replace(...)` — 항상 **원본** `my_string`에서 한 글자만 지운 결과를 `answer`에 덮어씀.
- 마지막 루프(`u` 제거) 결과만 살아남음.
- `"bus"`는 모음이 `u` 하나뿐이라 우연히 정답이 나온 것 — **샘플 통과 = 정답이 아님** (두수나눗셈에서 본 패턴과 같음).

**원인의 근본**: 문자열은 **불변(immutable)** 이라 `.replace()`가 원본을 수정하지 않고 **새 문자열을 반환**. 누적하려면 그 결과를 **자기 자신에게 다시 할당**해야 함.

**해결 방법**
```python
# 방법 A — my_string 자체를 갱신
def solution(my_string):
    for i in ["a","e","i","o","u"]:
        my_string = my_string.replace(i, "")   # ← 같은 변수에 재할당
    return my_string

# 방법 B — answer를 누적 변수로 (원본부터 시작)
def solution(my_string):
    answer = my_string
    for i in ["a","e","i","o","u"]:
        answer = answer.replace(i, "")         # ← answer에서 또 지움
    return answer

# 방법 C — 컴프리헨션 (가장 추천)
def solution(my_string):
    return ''.join(ch for ch in my_string if ch not in 'aeiou')

# 방법 D — str.translate (가장 빠름)
def solution(my_string):
    return my_string.translate(str.maketrans('', '', 'aeiou'))
```

**핵심 문법 / 배운 점**
- **`str.replace()`는 새 문자열 반환** (원본 그대로). 받아서 재할당해야 효과 있음.
- **누적/rebind 패턴**: `x = x.replace(...)` — 왼쪽과 오른쪽 변수명을 같게.
  - 비슷한 패턴: 뒤집한문자열의 `answer = ch + answer`
- **in-place 메서드와 새 객체 반환 메서드 비교**

  | 메서드 | 동작 | 반환값 |
  |---|---|---|
  | `arr.reverse()`, `arr.sort()` | 원본 수정 | `None` |
  | `s.replace()`, `sorted(arr)`, `reversed(arr)` | 원본 그대로 | **새 객체** |

  → 모르는 함수는 **반환값이 새 객체인지 / None인지** 항상 확인.
- **변수명 오타**는 NameError 안 나고 새 변수가 만들어짐 — 디버깅 어려움. 출력값이 안 바뀌면 오타부터 의심.
- **샘플 1개 통과는 우연**일 수 있다. 입력이 단순하면 정답으로 보일 뿐, 일반 케이스는 실패. 모음 2개 이상인 케이스로 직접 검증하는 습관.
- **컴프리헨션 + `''.join()`** 패턴은 "조건에 맞는 글자만 골라 합치기"의 정석.
  ```python
  ''.join(ch for ch in s if 조건)
  ```
