# 위상정렬 (Topological Sort)

## 한 줄 요약
**선후 관계가 있는 작업들을 순서대로 나열**하는 알고리즘. "X 끝나야 Y 시작 가능" 같은 의존성 그래프(DAG)에서 동작.

```
A → B → D
    ↓   ↑
    C ──┘

가능한 순서: A, B, C, D
또는:        A, B, C, D (이 경우 유일)
```

## 어디 쓰나

- **선수 과목 / 강의 순서**
- **빌드 의존성** (A.o 만들려면 A.c, B.h 필요)
- **작업 스케줄링** (X 끝나야 Y 시작)
- **컴파일러의 함수 호출 순서**
- 게임의 기술 트리

## 전제 조건 — DAG

**Directed Acyclic Graph** = 방향 + 사이클 없음.

사이클이 있으면 위상정렬 불가능 ("A 끝나야 B, B 끝나야 A" — 모순).

## 알고리즘 — Kahn's Algorithm (BFS 기반)

가장 흔하고 직관적. 큐 + indegree(들어오는 간선 수).

### 핵심 아이디어
1. **들어오는 간선이 0인 노드** = 선행 작업 없는 노드 → 바로 실행 가능
2. 그 노드를 결과에 추가하고, 그 노드가 가리키는 모든 노드의 indegree -1
3. 새로 indegree가 0이 된 노드 추가
4. 큐가 빌 때까지 반복

### 코드
```python
from collections import deque, defaultdict

def topological_sort(n, edges):
    graph = defaultdict(list)
    indegree = [0] * (n+1)

    for a, b in edges:        # a → b (a 끝나야 b 시작)
        graph[a].append(b)
        indegree[b] += 1

    q = deque()
    for i in range(1, n+1):
        if indegree[i] == 0:
            q.append(i)

    result = []
    while q:
        node = q.popleft()
        result.append(node)
        for nxt in graph[node]:
            indegree[nxt] -= 1
            if indegree[nxt] == 0:
                q.append(nxt)

    if len(result) != n:
        return None              # 사이클 있음
    return result
```

## 동작 추적

```
edges: (1,2), (1,3), (2,4), (3,4)

초기 indegree:
  1: 0  ← 시작 가능
  2: 1
  3: 1
  4: 2

큐: [1]    →   결과: [1]
1 처리 → 2,3의 indegree -1
  2: 0  ← 시작 가능
  3: 0  ← 시작 가능
큐: [2, 3]   →   결과: [1, 2]
2 처리 → 4의 indegree -1 → 4: 1
큐: [3]      →   결과: [1, 2, 3]
3 처리 → 4의 indegree -1 → 4: 0  ← 시작 가능
큐: [4]      →   결과: [1, 2, 3, 4] ✅
```

## 사이클 감지

위상정렬 결과의 길이가 **n과 다르면 사이클 존재**.
- 큐에 들어온 노드가 n보다 적음 = 어떤 노드는 indegree가 끝까지 0이 안 됨 = 사이클로 묶여 있음

```python

```

## DFS 기반 버전 (참고)

DFS를 끝까지 가서 **종료된 순서의 역순**이 위상정렬.
```python
visited = [False] * (n+1)
result = []

def dfs(node):
    visited[node] = True
    for nxt in graph[node]:
        if not visited[nxt]:
            dfs(nxt)
    result.append(node)

for i in range(1, n+1):
    if not visited[i]:
        dfs(i)
result.reverse()
```

코테에선 BFS 기반(Kahn)이 더 자주 쓰임 — 사이클 감지가 자연스러움.

## 시간복잡도

| | |
|---|---|
| 위상정렬 | O(V + E) |

V = 노드, E = 간선. 매우 빠름.

## 결과의 다양성

- 위상정렬은 **유일하지 않음** — 여러 가능한 순서 존재.
- "사전순으로 가장 빠른 순서"를 원하면 큐 대신 **우선순위 큐(`heapq`)** 사용.

```python
import heapq

heap = []
for i in range(1, n+1):
    if indegree[i] == 0:
        heapq.heappush(heap, i)

while heap:
    node = heapq.heappop(heap)
    result.append(node)
    for nxt in graph[node]:
        indegree[nxt] -= 1
        if indegree[nxt] == 0:
            heapq.heappush(heap, nxt)
```

## 신호 — 위상정렬 문제

- "**선후 관계**", "선행 조건", "X 끝나야 Y"
- "**작업 순서**", "스케줄"
- "**선수 과목**" (A 듣고 B)
- "의존성", "build order"
- 그래프 + 방향 + 순서

## 풀이 사고 순서

1. 그래프 만들기 (방향 그래프)
2. **indegree 배열** 초기화
3. indegree 0인 노드들 큐에 넣기
4. Kahn 알고리즘 돌리기
5. 결과 길이 확인 (사이클 검출)

## 흔한 실수

- **방향 헷갈림** — "A 끝나야 B" 면 `A → B`. 화살표 방향 명확히
- 무방향 그래프 추가 (양쪽 추가) — 위상정렬은 **방향만**
- 시작 노드 indegree 체크 누락 (모든 0인 노드 다 큐에 넣기)
- 사이클 검사 빼먹음

## 응용 패턴

### 1. 가장 늦은 작업 시간 (CPM)
- 각 작업에 시간이 있을 때 전체 끝나는 시간
- DP + 위상정렬

### 2. 강의 듣는 순서
- 선수 과목 그래프 → 위상정렬

### 3. 테이블 정렬 / 체인
- 부분 순서를 전체 순서로 변환

## 실전 팁

- 코드 거의 항상 같은 패턴 — 외워두면 빠름
- BFS 코드와 비슷하지만 **indegree 갱신**이 핵심
- 사이클 검출까지 한 번에 가능한 게 장점

## 대표 문제 유형

- 줄세우기, 음악프로그램 (백준)
- 게임 맵 최단거리 (그래프 + 위상)
