# 다익스트라 (Dijkstra) + 우선순위 큐

## 한 줄 요약
**가중치가 있는 그래프**에서 **한 점 → 모든 점 최단 거리**를 구하는 알고리즘. BFS의 일반화. 핵심은 **우선순위 큐(`heapq`)**.

## 왜 BFS로는 안 되나

BFS는 "간선 가중치가 1인 경우의 최단 거리"만 보장.
- 가중치가 다 다르면 (예: 도시 간 거리), BFS는 잘못된 답.
- 다익스트라는 **"지금까지 거리가 가장 작은 노드부터 처리"** 한다는 그리디 전략으로 해결.

```
A ─2─ B ─1─ C
│            │
└──── 4 ─────┘

BFS: A → C (1번 점프, 거리 4)
다익스트라: A → B → C (2번 점프, 거리 3) ⭐
```

## 우선순위 큐 (Priority Queue) 먼저

다익스트라의 핵심 도구. **항상 가장 작은(또는 큰) 값이 먼저 나오는 큐**.

파이썬에선 `heapq` 모듈로 구현 (최소 힙).

```python
import heapq

heap = []
heapq.heappush(heap, 5)
heapq.heappush(heap, 1)
heapq.heappush(heap, 3)

heapq.heappop(heap)    # 1 (항상 최솟값)
heapq.heappop(heap)    # 3
heapq.heappop(heap)    # 5
```

| 연산 | 시간 |
|---|---|
| heappush | O(log n) |
| heappop | O(log n) |
| heap[0] (최솟값 보기) | O(1) |

### 튜플로 우선순위 + 데이터
```python
heapq.heappush(heap, (거리, 노드))     # 거리 기준 정렬
dist, node = heapq.heappop(heap)
```

튜플은 첫 요소부터 비교하므로 **(우선순위, 데이터)** 형태가 표준.

### 최대 힙으로 쓰기
```python
heapq.heappush(heap, -value)
-heapq.heappop(heap)
```

## 다익스트라 알고리즘

```python
import heapq
from collections import defaultdict

def dijkstra(start, graph, n):
    INF = float('inf')
    dist = [INF] * (n+1)
    dist[start] = 0
    heap = [(0, start)]            # (거리, 노드)

    while heap:
        d, node = heapq.heappop(heap)
        if d > dist[node]:          # 이미 더 짧은 거리로 처리됨
            continue
        for nxt, w in graph[node]:
            new_dist = d + w
            if new_dist < dist[nxt]:
                dist[nxt] = new_dist
                heapq.heappush(heap, (new_dist, nxt))

    return dist
```

### 입력 형태
```python
# 엣지 리스트: (출발, 도착, 가중치)
edges = [(1,2,3), (1,3,5), (2,3,1), (2,4,2), (3,4,1)]

graph = defaultdict(list)
for a, b, w in edges:
    graph[a].append((b, w))
    # 무방향이면: graph[b].append((a, w))
```

## 동작 원리 (직관)

1. 시작점 거리를 0으로 두고 큐에 넣음
2. **현재까지 알려진 거리 중 가장 짧은 노드** 꺼냄
3. 그 노드의 이웃들에 대해 **"더 짧은 경로 발견되면 갱신"**
4. 갱신된 거리로 큐에 다시 넣음
5. 큐가 빌 때까지 반복

핵심 보장: **꺼낸 시점의 거리가 그 노드의 최종 최단 거리**.

## 시간복잡도

| | |
|---|---|
| 다익스트라 (heap 사용) | **O((V+E) log V)** |
| 다익스트라 (배열 검색) | O(V²) |
| BFS | O(V+E) — 가중치 1만 |

V = 노드, E = 간선. 보통 heap 버전이 빠름.

## 제약사항 ⚠️

- **음수 가중치 안 됨**. 음수 있으면 **벨만-포드** 사용.
- 그리디 전략이 깨짐 (음수 간선 통해 더 짧아질 수 있어서).

## BFS vs 다익스트라

| | BFS | 다익스트라 |
|---|---|---|
| 가중치 | 모두 1 (또는 동일) | 양수 가중치 |
| 자료구조 | 일반 큐 (`deque`) | 우선순위 큐 (`heapq`) |
| 시간 | O(V+E) | O((V+E) log V) |
| 보장 | 최단 (가중치 1) | 최단 (양수 가중치) |

→ "가중치 다 같으면 BFS, 다르면 다익스트라"

## 비슷한 알고리즘들

### 벨만-포드 (Bellman-Ford)
- 음수 가중치 OK
- 음수 사이클 검출 가능
- O(VE), 다익스트라보다 느림

### 플로이드-워셜 (Floyd-Warshall)
- **모든 쌍 최단 거리** (한 번에)
- O(V³)
- V가 작을 때 (~300) 코드가 짧고 강력

```python
INF = float('inf')
dist = [[INF]*n for _ in range(n)]
for i in range(n): dist[i][i] = 0
for a, b, w in edges: dist[a][b] = w

for k in range(n):           # 거쳐가는 점
    for i in range(n):
        for j in range(n):
            if dist[i][k] + dist[k][j] < dist[i][j]:
                dist[i][j] = dist[i][k] + dist[k][j]
```

3중 for문이지만 **3줄로 끝나는 게 매력**. V 작으면 다익스트라보다 코드 짧음.

## 우선순위 큐의 다른 활용

다익스트라 외에도 자주 쓰임.

### Top-K 문제
"가장 큰 K개" / "가장 작은 K개"
```python
heapq.nlargest(3, arr)         # [9, 7, 5] (정렬됨)
heapq.nsmallest(3, arr)        # [1, 2, 3]
```

### 작업 스케줄링
"가장 빨리 끝나는 작업부터 처리" — heap에 (완료시간, 작업) 튜플로.

### 병합 정렬된 K개 리스트
```python
import heapq
list(heapq.merge(arr1, arr2, arr3))    # 정렬된 결과
```

## 풀이 사고 순서

1. 그래프 문제인가? 가중치 있나?
2. 가중치 양수? → 다익스트라
3. 음수 있음? → 벨만-포드
4. 모든 쌍? → 플로이드-워셜 (V 작을 때)
5. 그래프 만들기 + 다익스트라 코드 적용

## 신호 — 다익스트라 문제

- "**최단 거리**" + 가중치 (시간/비용/거리)
- "**최소 비용**으로 도달"
- 도시, 도로, 네트워크
- 한 점에서 모든 점 / 한 점에서 한 점

## 실전 팁

- **다익스트라 코드 통째로 외워두기** — 항상 같은 패턴
- `dist[node] = INF`로 초기화 (`float('inf')`)
- "이미 처리된 거 또 꺼내지 마" 체크 (`if d > dist[node]: continue`)
- 방향 그래프인지 무방향인지 확인 (양쪽 추가 여부)

## 대표 문제 유형

- 배달, 가장 먼 노드 (프로그래머스)
- 최단경로, 미로 탈출, 특정한 최단경로 (백준)
