# 이분탐색 (Binary Search)

## 한 줄 요약
**정렬된 데이터에서 절반씩 잘라가며** 원하는 값을 찾는 방법. n개 중에서 찾는 데 **O(log n)** — 100만 개도 20번이면 끝남.

## 직관

전화번호부에서 "김철수" 찾기. 처음부터 한 장씩 보지 않고:
1. 책 가운데 펼침 → "박**" 나옴
2. "김"은 "박"보다 앞이니 앞쪽 절반만 봄
3. 또 가운데 펼침 → "구**" 나옴
4. "김"은 "구"보다 뒤니 뒤쪽 절반만 봄
5. ...반복

**매번 후보가 절반으로 줄어듦** → log₂(n)번이면 1개로 좁혀짐.

## 기본 코드

```python
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1   # 못 찾음
```

⚠️ 전제 조건: **`arr`는 정렬되어 있어야 함**.

## bisect 모듈 (파이썬 표준)

직접 짜지 말고 `bisect`를 쓰면 끝.

```python
import bisect

arr = [1, 3, 5, 5, 5, 7, 9]

bisect.bisect_left(arr, 5)    # 2 (5가 들어갈 가장 왼쪽 인덱스)
bisect.bisect_right(arr, 5)   # 5 (5의 오른쪽)
bisect.bisect(arr, 5)         # bisect_right와 동일

# 정렬 유지하며 삽입
bisect.insort(arr, 4)         # arr = [1,3,4,5,5,5,7,9]
```

### bisect로 존재 여부 확인
```python
def exists(arr, target):
    i = bisect.bisect_left(arr, target)
    return i < len(arr) and arr[i] == target
```

### 범위 카운트
```python
# arr에서 [lo, hi] 범위 원소 개수
def count_in_range(arr, lo, hi):
    return bisect.bisect_right(arr, hi) - bisect.bisect_left(arr, lo)
```

## 매개변수 탐색 (Parametric Search) ⭐

**이분탐색의 진짜 위력**. "값을 찾는 게 아니라, **답이 될 수 있는 범위를 좁히는** 데 사용".

### 패턴
"X가 가능한가?" 라는 결정 문제로 바꿀 수 있고, 그 답이 **단조성**을 가지면 이분탐색.

```
X=1 → 가능
X=2 → 가능
X=3 → 가능
X=4 → 불가능
X=5 → 불가능
        ↑ 경계
```

→ 이분탐색으로 경계 찾기. **가능한 X 중 최댓값** = 3.

### 예: "k개 잘라서 만들 수 있는 최대 길이"

> 떡 길이의 합이 M 이상이 되도록 떡을 자를 때, 가능한 최대 절단기 높이는?

```python
def solve(lengths, target):
    left, right = 0, max(lengths)
    answer = 0
    while left <= right:
        mid = (left + right) // 2
        total = sum(max(0, x - mid) for x in lengths)  # 이 높이로 자르면 얻는 양
        if total >= target:
            answer = mid           # 가능 → 더 높여보기
            left = mid + 1
        else:
            right = mid - 1        # 불가능 → 낮춰야
    return answer
```

핵심: "**가능 vs 불가능**"의 경계를 이분탐색으로 찾기.

### 매개변수 탐색 신호
- "**최대 X**의 값", "**최소 X**의 값"을 구하라
- X에 대해 "이게 가능한가?" 가 비교적 쉽게 판단됨
- X가 커질수록(작아질수록) 단조롭게 가능/불가능이 바뀜

## LIS (최장 증가 부분수열)

이분탐색의 또 다른 응용. O(n log n)으로 LIS 길이 구하기.

```python
import bisect

def lis_length(arr):
    tails = []
    for x in arr:
        i = bisect.bisect_left(tails, x)
        if i == len(tails):
            tails.append(x)
        else:
            tails[i] = x
    return len(tails)
```

`tails[i]` = "길이 i+1인 증가 부분수열들 중 끝값의 최솟값". 직관적이진 않지만 외워두면 강력.

## 시간복잡도

| 연산 | 일반 탐색 | 이분탐색 |
|---|---|---|
| 검색 | O(n) | **O(log n)** |
| 삽입 (정렬 유지) | O(n) | O(n) — 옮기는 비용 |

n=10⁶이면:
- 일반 탐색: 100만 번
- 이분탐색: ~20번

## 신호 — 이분탐색 문제

- "**정렬된 배열**에서 검색"
- "**최대/최소 X**를 구하라" (매개변수 탐색)
- 답의 범위가 명확함 (1~10⁹ 이런 식)
- "X가 가능한가?" 가 O(n)으로 판단 가능
- 완탐으로는 시간 초과 (10⁹ 같은 큰 범위)

## 풀이 사고 순서

### 일반 이분탐색
1. 정렬되어 있나? 안 되어 있으면 정렬 먼저 (`sorted()`)
2. `bisect_left` / `bisect_right` 중 어느 게 맞나
3. 끝

### 매개변수 탐색
1. **답을 X**라고 두고, "X가 가능한가?" 판정 함수 만들기
2. X 범위 정하기 (`left`, `right`)
3. 단조성 확인 (X가 커지면 점점 가능 → 불가능, 또는 반대)
4. 이분탐색으로 경계 찾기

## 흔한 실수

- **정렬 안 한 배열**에 이분탐색 → 잘못된 답
- `mid = (left + right) // 2` 에서 **오버플로우**는 파이썬에선 걱정 X
- `left <= right` vs `left < right` — 패턴마다 다름. 헷갈리면 한 패턴으로 통일
- 매개변수 탐색에서 답 갱신 위치 (`answer = mid`) 빠뜨림
- 무한 루프 — `left = mid` 같이 안 줄어드는 갱신 (반드시 `mid+1` 또는 `mid-1`)

## 실전 팁

- 단순 검색은 **`bisect`** 한 줄로 끝
- 매개변수 탐색은 외울 만한 패턴 — 익혀두면 어려운 문제 풀이의 무기
- 답이 실수면 `while right - left > 1e-9` 같이 정밀도 기반
- 항상 **단조성 확인**부터

## 대표 문제 유형

- 입국심사, 징검다리, 예산 (프로그래머스)
- 랜선 자르기, 나무 자르기 (백준)
- 정렬된 배열에서 특정 값 검색
