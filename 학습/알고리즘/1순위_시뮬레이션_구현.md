# 시뮬레이션 / 구현

## 한 줄 요약
**문제에서 요구하는 절차를 그대로 코드로 옮기는** 유형. "이 규칙대로 흘러가는 걸 시뮬레이션해라"가 핵심. 알고리즘이라기보다 **꼼꼼함과 정리력 테스트**.

## 특징

- 지문이 길고 규칙이 많음
- 알고리즘 자체는 단순 — 그냥 시키는 대로 하면 됨
- 인덱스, 좌표, 방향 처리에서 실수 나옴
- "이거 어떤 알고리즘이지?" 고민할 필요 없이 **순서도대로 짜면 됨**

## 핵심 패턴

### 1. 격자(grid)에서 방향 이동

```python
# 상하좌우
dx = [-1, 1, 0, 0]
dy = [0, 0, -1, 1]

x, y = 5, 5
for d in range(4):
    nx, ny = x + dx[d], y + dy[d]
    if 0 <= nx < n and 0 <= ny < m:
        # 처리
        ...
```

8방향이면 `dx = [-1,-1,-1,0,0,1,1,1], dy = [-1,0,1,-1,1,-1,0,1]`.

### 2. 회전 / 방향 전환

방향을 0~3으로 정의하고 % 연산:
```python
# 0:상, 1:우, 2:하, 3:좌
direction = 0
direction = (direction + 1) % 4    # 시계방향 90도
direction = (direction - 1) % 4    # 반시계 90도
```

### 3. 시간 흐름 시뮬레이션

```python
time = 0
while not finished:
    # 시간 단위로 일어나는 일들 처리
    time += 1
```

### 4. 상태 갱신

각 단계마다 상태가 바뀜. **새 상태를 기존 위에 덮어쓰면 같은 단계 안에서 영향 받음** → 새 배열에 따로 만들기.
```python
# ❌ 한 칸 처리하다가 옆 칸에 영향
for x in range(n):
    grid[x] = new_value(...)

# ✅ 새 배열에 결과 만들고 한 번에 교체
new_grid = [[0]*m for _ in range(n)]
for x in range(n):
    for y in range(m):
        new_grid[x][y] = compute(grid, x, y)
grid = new_grid
```

## 자주 쓰는 도구

```python
# 격자 만들기
grid = [[0]*m for _ in range(n)]    # ❌ [[0]*m]*n 절대 금지

# 좌표는 튜플로
(x, y), 또는 deque에 (x, y, time) 같이

# 회전
direction = (direction + 1) % 4

# 거리
abs(x1-x2) + abs(y1-y2)             # 맨해튼 거리
((x1-x2)**2 + (y1-y2)**2) ** 0.5    # 유클리드

# 행렬 회전 (90도 시계)
rotated = list(zip(*matrix[::-1]))
```

## 풀이 사고 순서

1. **지문 한 번 천천히 읽기** — 절대 빨리 읽지 말 것
2. **변하는 상태가 뭔지 정리** (좌표, 방향, 시간, 점수, ...)
3. **각 단계에서 무슨 일이 일어나는지** 한국어로 정리
4. **자료구조 정하기** — 격자면 2D 리스트, 좌표 모음이면 set
5. **코드로 옮기기** — 변수명 명확히
6. **샘플 손으로 추적** — 1단계, 2단계 직접 따라가보기

## 디버깅 팁

- **격자 출력 함수** 미리 만들어두기
  ```python
  def print_grid(g):
      for row in g:
          print(row)
      print()
  ```
- 매 단계 출력해서 의도대로 흘러가는지 눈으로 확인
- 인덱스 헷갈리면 작은 케이스 (3×3)로 손으로 시뮬

## 신호 — 시뮬레이션/구현 문제

- 지문이 1쪽 이상으로 길다
- "다음 규칙에 따라 ~를 시뮬레이션하시오"
- 격자, 좌표, 방향, 시간 단위
- "k번 반복한 후의 상태를 구하라"
- 딱히 알고리즘 이름이 안 떠오름 — 그냥 따라하면 될 것 같음

## 흔한 실수

- **`[[0]*m]*n`** ← 모든 행이 같은 객체 참조됨. 반드시 `[[0]*m for _ in range(n)]`
- **인덱스 범위 체크 누락** — `0 <= nx < n` 빠뜨리기
- **상태 덮어쓰기** — 한 단계 안에서 옆 칸이 이미 갱신된 값 보게 됨
- **방향 정의 헷갈림** — (행, 열) vs (x, y) 혼동
- **종료 조건 누락** — 무한 루프

## 실전 팁

- 변수명 길어도 OK — `current_direction` 이 `cd`보다 나음
- 함수로 쪼개기 — `def move(x, y, d)`, `def rotate(d)` 등
- 디버깅 출력 적극 활용 (제출 전엔 지우거나 주석 처리)
- 시간복잡도는 보통 여유 있음 — **정확성** 우선

## 대표 문제 유형 (프로그래머스/백준)

- 자물쇠와 열쇠
- 카카오프렌즈 컬러링북
- 배달, 청소, 이동, 삼각 달팽이
- 거리두기 확인하기
- 연구소, 미세먼지 안녕! (시뮬레이션 + BFS)
