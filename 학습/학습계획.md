# 코테 학습 계획

자료구조와 알고리즘을 어떤 순서로, 왜 그 순서로 공부할지 정리한 로드맵.

> 함께 보는 문서
> - **공부 가이드**: `../코테_공부_가이드.md` — 사고법/체크리스트
> - **문법 사전**: `../파이썬_문법_레퍼런스.md` — 내장함수/도구

## 우선순위 정리

### 1순위 — 통과를 위한 필수
**시뮬레이션/구현 → 완전 탐색 → BFS/DFS**

기반 자료구조: 스택, 큐, 링크드리스트, 해시테이블, 그래프, 트리

→ 이것만 완벽해도 일반 코테는 통과 가능.

### 2순위 — 고득점을 위한 추가
**DP → 다익스트라/우선순위 큐 → 이분탐색 → 위상정렬**

→ 어려운 문제, 큰 입력에서 차이를 만듦.

---

## 의존성 그래프 (학습 순서가 중요한 이유)

```
[자료구조 기반]
  스택 ─────┐
  큐 ───────┼──→ BFS/DFS ──┐
  해시 ──┐  │              │
  그래프 ┘  │              ├──→ 다익스트라
  트리 ─────┘              │   (BFS+우선순위큐+그래프 필요)
                           │
[알고리즘]                 │
  시뮬/구현                │
  완전탐색 ─→ DP           │
                           │
  이분탐색 (독립)          │
  위상정렬 ←───────────────┘
                (그래프 필요)
```

**핵심 의존성**:
- **DP** 하려면 **완전탐색**을 먼저 알아야 함 (DP는 "중복되는 부분 문제 메모이제이션"이라, 완전탐색 사고가 기반)
- **다익스트라**는 **BFS + 큐 + 그래프**의 조합이라 이 셋을 먼저
- **위상정렬**도 그래프 위에서 동작하므로 그래프 학습 필수

## SQL은 별개 트랙

자료구조/알고리즘과 무관하게 병행 가능. 백엔드/데이터 직군이면 필수.

| 일 | 학습 |
|---|---|
| 1 | 기본 문법 (SELECT/WHERE/ORDER BY) |
| 2 | 조인 (INNER/LEFT) |
| 3 | 집계 (GROUP BY/HAVING) |
| 4 | 서브쿼리 |
| 5 | 윈도우 함수 |

프로그래머스 SQL 코딩테스트 LV1~LV4로 실전 연습.

## 학습 원칙

### 한 번은 직접 구현
- 스택, 큐, 링크드리스트는 **클래스로 직접 짜보기** (한 번이면 충분)
- 정렬 알고리즘 (버블/선택/퀵/병합) 한 번씩
- BFS/DFS 두 패턴 직접 짜보기
- → 그 다음부터는 내장함수 (`deque`, `sorted`, `heapq`)

### 문서 → 문제 → 회고 사이클
1. 학습 문서 1편 정독 (10~15분)
2. 관련 문제 1~2개 풀이 (30분~1시간)
3. 막힌 부분/배운 점 → `programmers/LV{N}/문제별_회고.md` 에 기록

### 복습 주기
- 학습 다음 날 다시 보기
- 1주일 후 다시 보기
- 1달 후 회고 노트 통째로 다시 읽기 → 본인의 성장이 보임
