# 서브쿼리 (Subquery)

## 한 줄 요약
**쿼리 안에 또 쿼리를 넣는** 기법. "평균보다 많이 받는 사람"처럼 **다른 쿼리 결과를 조건/값으로 사용**할 때.

## 직관

"평균 연봉보다 많이 받는 사원" 을 구하려면:
1. 먼저 평균 연봉을 알아야 함
2. 그 값보다 큰 사람을 골라냄

```sql
-- 1. 평균 알아내기
SELECT AVG(salary) FROM employees;   -- 5500

-- 2. 그 값보다 큰 사람
SELECT * FROM employees WHERE salary > 5500;

-- 한 번에:
SELECT * FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);
```

## 위치별 분류

서브쿼리는 어디에 들어가는지에 따라 명칭이 다름.

### 1. WHERE 절 — 조건으로
```sql
SELECT * FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);
```

### 2. SELECT 절 — 값으로 (스칼라 서브쿼리)
한 행/한 컬럼 반환해야 함.
```sql
SELECT name,
       salary,
       (SELECT AVG(salary) FROM employees) AS avg_sal
FROM employees;
```

### 3. FROM 절 — 가상 테이블로 (인라인 뷰)
서브쿼리 결과를 테이블처럼 사용.
```sql
SELECT dept_id, max_sal
FROM (
    SELECT dept_id, MAX(salary) AS max_sal
    FROM employees
    GROUP BY dept_id
) AS dept_max
WHERE max_sal > 5000;
```

### 4. HAVING 절 — 그룹 필터
```sql
SELECT dept_id, COUNT(*)
FROM employees
GROUP BY dept_id
HAVING COUNT(*) > (SELECT AVG(cnt)
                   FROM (SELECT COUNT(*) AS cnt
                         FROM employees
                         GROUP BY dept_id) t);
```

## 서브쿼리 결과 형태에 따른 분류

### 1. 단일 값 (스칼라)
한 행, 한 컬럼.
```sql
WHERE salary > (SELECT AVG(salary) FROM employees);
```
`= > <` 등 일반 비교 가능.

### 2. 다중 값 (단일 컬럼)
여러 행, 한 컬럼.
```sql
-- IN 연산자
WHERE dept_id IN (SELECT dept_id FROM departments WHERE region = 'US');

-- ANY / ALL
WHERE salary > ANY (SELECT salary FROM ...)    -- 하나라도 더 크면
WHERE salary > ALL (SELECT salary FROM ...)    -- 모두보다 더 크면
```

### 3. 다중 행, 다중 컬럼
```sql
WHERE (dept_id, position) IN (
    SELECT dept_id, position FROM ... 
);
```

## EXISTS — "있는지 검사"

서브쿼리 결과 **존재 여부만 확인**.

```sql
-- 주문이 1건이라도 있는 사용자
SELECT * FROM users u
WHERE EXISTS (
    SELECT 1 FROM orders o WHERE o.user_id = u.id
);

-- 주문이 없는 사용자
SELECT * FROM users u
WHERE NOT EXISTS (
    SELECT 1 FROM orders o WHERE o.user_id = u.id
);
```

`SELECT 1`은 관례 — EXISTS는 컬럼 안 보고 존재만 체크하므로.

### IN vs EXISTS

| | 동작 | 언제 |
|---|---|---|
| `IN (서브쿼리)` | 값 비교 | 결과 작을 때 |
| `EXISTS (서브쿼리)` | 존재 확인 | 결과 클 때, NULL 안전 |

```sql
-- 동일한 결과 (대부분)
WHERE dept_id IN (SELECT dept_id FROM departments WHERE ...)
WHERE EXISTS (SELECT 1 FROM departments d WHERE d.dept_id = e.dept_id AND ...)
```

## 상관 서브쿼리 (Correlated Subquery)

**바깥 쿼리의 값을 안에서 참조**하는 서브쿼리. 매 행마다 실행됨.

```sql
-- 같은 부서 평균보다 많이 받는 사원
SELECT *
FROM employees e
WHERE salary > (
    SELECT AVG(salary)
    FROM employees
    WHERE dept_id = e.dept_id     -- ⭐ 바깥 e 참조
);
```

각 사원의 **자기 부서 평균**과 비교. 매 행마다 서브쿼리가 다른 결과.

⚠️ 데이터 많으면 느림 (n번 실행됨). JOIN으로 바꿀 수 있으면 그게 빠름.

## 서브쿼리 vs 조인

같은 결과를 둘 다로 표현 가능한 경우 많음.

### 예: 부서가 'IT'인 사원

#### 서브쿼리
```sql
SELECT * FROM employees
WHERE dept_id IN (
    SELECT dept_id FROM departments WHERE name = 'IT'
);
```

#### 조인
```sql
SELECT e.*
FROM employees e
INNER JOIN departments d ON e.dept_id = d.dept_id
WHERE d.name = 'IT';
```

| | 서브쿼리 | 조인 |
|---|---|---|
| 가독성 | 단계별로 명확 | 한 번에 보기 |
| 성능 | DBMS 따라 다름 | 보통 빠름 |
| 컬럼 사용 | 결과만 | 양쪽 컬럼 다 |

→ 양쪽 다 써보고 익숙해지기. 코테는 가독성 좋은 쪽으로.

## 자주 쓰는 패턴

### 1. 최댓값 가진 행
```sql
-- 가장 비싼 상품 (값만 X, 행 자체)
SELECT * FROM products
WHERE price = (SELECT MAX(price) FROM products);
```

### 2. 그룹별 최댓값 가진 행
```sql
-- 부서별 가장 연봉 높은 사람
SELECT * FROM employees e
WHERE salary = (
    SELECT MAX(salary)
    FROM employees
    WHERE dept_id = e.dept_id
);
```

### 3. ~보다 X% 많은
```sql
SELECT * FROM employees
WHERE salary > 1.5 * (SELECT AVG(salary) FROM employees);
```

### 4. 카테고리별 통계 합치기
```sql
SELECT
    e.name,
    e.salary,
    (SELECT AVG(salary) FROM employees WHERE dept_id = e.dept_id) AS dept_avg
FROM employees e;
```

### 5. ~없는 사용자
```sql
-- 주문 한 번도 안 한 사용자
SELECT * FROM users
WHERE id NOT IN (SELECT DISTINCT user_id FROM orders WHERE user_id IS NOT NULL);

-- 또는 EXISTS
SELECT * FROM users u
WHERE NOT EXISTS (SELECT 1 FROM orders WHERE user_id = u.id);

-- 또는 LEFT JOIN
SELECT u.* FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE o.id IS NULL;
```

## NULL 함정 ⚠️

`NOT IN` + NULL은 위험.
```sql
-- ❌ 서브쿼리에 NULL 있으면 결과 비어버림
WHERE id NOT IN (SELECT user_id FROM orders);

-- ✅
WHERE id NOT IN (SELECT user_id FROM orders WHERE user_id IS NOT NULL);

-- ✅ 또는 EXISTS / LEFT JOIN
```

이유: `NULL != X` 의 결과가 `UNKNOWN`이라 NOT IN이 항상 false 처리.

## CTE (Common Table Expression) — `WITH`

서브쿼리에 이름 붙여서 재사용.

```sql
WITH dept_avg AS (
    SELECT dept_id, AVG(salary) AS avg_sal
    FROM employees
    GROUP BY dept_id
)
SELECT e.name, e.salary, da.avg_sal
FROM employees e
JOIN dept_avg da ON e.dept_id = da.dept_id;
```

복잡한 쿼리를 단계별로 쪼개서 읽기 쉽게. 코테에서도 점점 자주 보임.

## 풀이 사고 순서

1. "다른 쿼리 결과가 필요" → 서브쿼리 / CTE
2. **단일 값**? → 비교 연산자 (`= > < `)
3. **여러 값**? → IN, EXISTS, ANY, ALL
4. 서브쿼리 vs 조인 — 둘 다 가능하면 간단한 쪽

## 흔한 실수

- 단일 값 자리에 여러 행 반환 → 에러
- `NOT IN` + NULL 함정
- 상관 서브쿼리가 너무 많이 돌아 느림
- FROM 절 서브쿼리에 별칭 안 줌 → 에러

## 신호 — 서브쿼리 문제

- "**평균보다**", "**최댓값**", "**가장 ~한 X**"
- "다른 테이블에 ~ 있는/없는"
- 다단계 계산 (먼저 X 구하고 그걸로 Y)

## 실전 팁

- 가능하면 **CTE(`WITH`)** 로 단계 나누기 (가독성 ↑)
- 같은 쿼리 두 번 쓰지 말고 CTE로 한 번만
- 상관 서브쿼리는 신중히 (성능)
- IN/EXISTS 어느 게 더 빠른지는 실측

## 대표 문제 유형 (프로그래머스)

- 가격이 제일 비싼 식품, 평균 일일 대여 요금
- 즐겨찾기가 가장 많은 식당, 헤비 유저가 소유한 장소
- 노선별 평균 운행 시간 (CTE 활용)
