# 집계 함수 + GROUP BY

## 한 줄 요약
**여러 행을 하나의 결과로 요약**하는 함수들. "총 몇 명?", "평균 연봉?", "부서별 인원 수?" 같은 질문에 답하는 도구.

## 기본 집계 함수

```sql
COUNT(*)              -- 행 수 (NULL 포함)
COUNT(컬럼)            -- NULL 제외한 개수
COUNT(DISTINCT 컬럼)   -- 중복 제외 개수

SUM(컬럼)              -- 합계 (숫자만)
AVG(컬럼)              -- 평균
MIN(컬럼)              -- 최솟값
MAX(컬럼)              -- 최댓값
```

### 예
```sql
SELECT COUNT(*), AVG(salary), MAX(salary)
FROM employees;
```

```
COUNT(*)   AVG(salary)   MAX(salary)
100        5500          12000
```

## GROUP BY — 그룹별 집계 ⭐

전체가 아닌 **카테고리별 요약**.

```sql
-- 부서별 인원 수
SELECT dept_id, COUNT(*) AS cnt
FROM employees
GROUP BY dept_id;
```

```
dept_id   cnt
1         30
2         25
3         45
```

### 동작 원리
```
원본:                       그룹화 후:
id  dept_id  salary         dept_id  [salary들]
1   1        5000           1        [5000, 6000, ...]
2   1        6000           2        [4000, 7000, ...]
3   2        4000           3        [...]
4   2        7000
...
```

각 그룹마다 집계 함수 적용.

### 다중 컬럼 그룹화
```sql
SELECT dept_id, position, COUNT(*) AS cnt, AVG(salary) AS avg_sal
FROM employees
GROUP BY dept_id, position;
```

(부서, 직급) 조합별로 그룹.

## HAVING — 그룹 후 필터링

`WHERE`는 그룹 **전**, `HAVING`은 그룹 **후**.

```sql
SELECT dept_id, COUNT(*) AS cnt
FROM employees
WHERE salary > 3000              -- 행 단위 필터 (그룹 전)
GROUP BY dept_id
HAVING COUNT(*) >= 10;           -- 그룹 단위 필터 (그룹 후)
```

→ "salary 3000 넘는 사원만 골라서, 부서별로 묶고, 그 중 인원 10명 이상인 부서만"

### WHERE vs HAVING

| | WHERE | HAVING |
|---|---|---|
| 적용 시점 | 그룹화 **전** (개별 행) | 그룹화 **후** |
| 집계 함수 사용 | 불가 | **가능** ⭐ |
| 일반 컬럼 | 가능 | 가능 (GROUP BY 컬럼만) |

```sql
WHERE COUNT(*) > 5      -- ❌ 불가 (집계 함수)
HAVING COUNT(*) > 5     -- ✅ 가능
```

## SELECT 가능한 컬럼 규칙 ⭐

GROUP BY 사용 시 SELECT엔 다음만 올 수 있음:
- **GROUP BY에 적은 컬럼** (그룹화 키)
- **집계 함수의 결과**

```sql
-- ❌ name은 그룹마다 여러 개 → 뭘 보여야 할지 모호
SELECT dept_id, name, COUNT(*)
FROM employees
GROUP BY dept_id;

-- ✅
SELECT dept_id, COUNT(*)
FROM employees
GROUP BY dept_id;
```

(MySQL은 비표준 허용하기도 하지만, 표준 SQL과 코테 채점은 엄격하게 막음)

## 자주 쓰는 패턴

### 1. 카테고리별 카운트
```sql
SELECT category, COUNT(*) AS cnt
FROM products
GROUP BY category
ORDER BY cnt DESC;
```

### 2. 카테고리별 합/평균
```sql
SELECT user_id, SUM(amount) AS total
FROM orders
GROUP BY user_id;
```

### 3. 가장 많이/적게 (TOP-1)
```sql
SELECT category, COUNT(*) AS cnt
FROM products
GROUP BY category
ORDER BY cnt DESC
LIMIT 1;
```

### 4. 중복 찾기
```sql
SELECT email, COUNT(*) AS cnt
FROM users
GROUP BY email
HAVING COUNT(*) > 1;
```

### 5. 조건부 카운트 (CASE WHEN)
```sql
SELECT
    SUM(CASE WHEN gender = 'M' THEN 1 ELSE 0 END) AS male,
    SUM(CASE WHEN gender = 'F' THEN 1 ELSE 0 END) AS female
FROM users;
```

또는 더 깔끔히:
```sql
SELECT
    COUNT(IF(gender = 'M', 1, NULL)) AS male,
    COUNT(IF(gender = 'F', 1, NULL)) AS female
FROM users;
```

### 6. 날짜별 집계
```sql
SELECT DATE_FORMAT(created_at, '%Y-%m') AS month, COUNT(*)
FROM orders
GROUP BY DATE_FORMAT(created_at, '%Y-%m')
ORDER BY month;
```

## NULL 처리

집계 함수는 **NULL을 자동으로 무시** (COUNT(*) 빼고).

```sql
-- 5명 중 3명만 salary 있음
SELECT
    COUNT(*),         -- 5
    COUNT(salary),    -- 3
    AVG(salary)       -- 3명의 평균
FROM employees;
```

`COUNT(*)`는 행 자체를 셈, `COUNT(컬럼)`은 NULL 제외.

## 집계 + 조인 패턴

```sql
-- 사용자별 주문 수와 총액
SELECT u.id, u.name, COUNT(o.id) AS order_count, SUM(o.amount) AS total
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
GROUP BY u.id, u.name;
```

LEFT JOIN으로 주문 없는 사용자도 포함, 그러면 `COUNT(o.id)`는 0이 됨 (NULL은 안 셈).

## 풀이 사고 순서

1. "**카테고리별 ~**", "**~당 ~**" 같은 표현 → GROUP BY
2. 그룹화 기준 컬럼 정하기
3. 어떤 집계 함수 (COUNT/SUM/AVG/MIN/MAX)
4. 그룹 후 필터링 필요? → HAVING
5. 정렬 + LIMIT

## 실행 순서 다시

```
FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT
```

GROUP BY가 SELECT보다 먼저 처리됨 → SELECT의 별칭은 GROUP BY/HAVING에서 못 씀(MySQL 일부는 허용).

## 흔한 실수

- SELECT에 그룹화/집계 안 된 컬럼 → 에러 (또는 비결정적 결과)
- HAVING 자리에 WHERE 쓰기 (집계 함수 못 들어감)
- COUNT(*) vs COUNT(컬럼) — NULL 차이
- 그룹화 후 정렬 빠뜨림

## 신호 — 집계/GROUP BY 문제

- "**~의 개수**", "**~별 평균**", "**총합**"
- "가장 많은", "가장 적은" + 카테고리
- "중복", "유일한"
- "X 이상인 그룹"

## 실전 팁

- COUNT(*) 가 가장 자주 씀
- 그룹별 TOP-K는 보통 윈도우 함수로 (다음 문서)
- HAVING과 ORDER BY에선 SELECT 별칭 사용 가능
- 코테에선 `LIMIT 1`로 1위 찾는 패턴 자주 나옴

## 대표 문제 유형 (프로그래머스)

- 동물 수 구하기, 이름에 el이 들어가는 동물
- 카테고리별 도서 판매량, 상품을 구매한 회원 비율
- 진료과별 총 예약 횟수 출력하기
