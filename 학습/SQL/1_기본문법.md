# SQL 기본 문법

## 한 줄 요약
SQL = **Structured Query Language**. 데이터베이스에 "이런 데이터 줘"라고 묻는 언어. 코테 SQL은 거의 **SELECT**만 잘하면 됨.

## SELECT — 데이터 가져오기

기본 형태:
```sql
SELECT 컬럼들
FROM 테이블
WHERE 조건
ORDER BY 정렬기준
LIMIT 개수;
```

### 예: 사원 정보
```sql
SELECT name, salary
FROM employees
WHERE salary >= 5000
ORDER BY salary DESC
LIMIT 10;
```

"employees 테이블에서 salary가 5000 이상인 사람들의 name과 salary를 salary 내림차순으로 10명만 줘".

## SELECT 자세히

### 컬럼 선택
```sql
SELECT *                    -- 모든 컬럼
SELECT name                 -- 특정 하나
SELECT name, age, salary    -- 여러 개

SELECT DISTINCT department  -- 중복 제거
```

### 별칭 (AS)
```sql
SELECT name AS 이름, salary AS 연봉
FROM employees;

-- AS 생략 가능
SELECT name 이름, salary 연봉 FROM employees;
```

### 계산식
```sql
SELECT name, salary * 12 AS annual_salary
FROM employees;
```

## WHERE — 조건 필터링

### 비교 연산자
```sql
WHERE age = 30           -- 같음 (== 아님!)
WHERE age != 30          -- 다름 (또는 <>)
WHERE age >= 30
WHERE age < 30
```

### 논리 연산자
```sql
WHERE age >= 30 AND salary > 5000
WHERE department = 'IT' OR department = 'HR'
WHERE NOT department = 'HR'
```

### 범위
```sql
WHERE age BETWEEN 30 AND 40         -- 30 <= age <= 40 (양끝 포함)
WHERE age NOT BETWEEN 30 AND 40
```

### 목록
```sql
WHERE department IN ('IT', 'HR', 'Sales')
WHERE department NOT IN ('IT')
```

### 패턴 매칭 (LIKE)
```sql
WHERE name LIKE 'Kim%'      -- Kim으로 시작 (% = 0개 이상 글자)
WHERE name LIKE '%Lee'      -- Lee로 끝
WHERE name LIKE '%Park%'    -- Park 포함
WHERE name LIKE 'K_m'       -- K?m (_ = 정확히 1글자)
```

### NULL 처리
```sql
WHERE manager_id IS NULL
WHERE manager_id IS NOT NULL

-- 주의: = NULL 은 안 됨!
WHERE manager_id = NULL    -- ❌ 항상 false
```

## ORDER BY — 정렬

```sql
ORDER BY salary             -- 오름차순 (기본)
ORDER BY salary ASC         -- 명시적 오름차순
ORDER BY salary DESC        -- 내림차순

-- 다중 정렬
ORDER BY department ASC, salary DESC
-- department 오름, 같으면 salary 내림
```

## LIMIT — 개수 제한

```sql
LIMIT 10               -- 처음 10개
LIMIT 10 OFFSET 20     -- 20개 건너뛴 후 10개 (페이지네이션)
```

⚠️ DBMS마다 다름 (MySQL/SQLite는 LIMIT, Oracle은 ROWNUM, MSSQL은 TOP). 프로그래머스는 보통 MySQL 기준.

## NULL 처리 함수

```sql
IFNULL(컬럼, 대체값)        -- NULL이면 대체값
COALESCE(컬럼1, 컬럼2, ...)  -- 첫 번째 non-NULL 반환
```

```sql
SELECT IFNULL(name, 'Unknown') FROM users;
```

## 문자열 함수

```sql
UPPER(s), LOWER(s)
LENGTH(s)                       -- 글자 수
CONCAT(s1, s2, ...)            -- 이어붙이기
SUBSTRING(s, start, length)    -- 부분 추출 (1부터 시작!)
REPLACE(s, old, new)
TRIM(s)                         -- 양끝 공백 제거

-- MySQL: || 대신 CONCAT 사용
SELECT CONCAT(first, ' ', last) FROM users;
```

## 날짜 함수 (MySQL 기준)

```sql
NOW()                          -- 현재 시각
CURDATE()                      -- 오늘 날짜

YEAR(date), MONTH(date), DAY(date)
DATE_FORMAT(date, '%Y-%m-%d')  -- 포맷
DATEDIFF(d1, d2)               -- 일수 차이

-- 자주 쓰는 포맷 코드
-- %Y: 4자리 연도, %m: 월(01-12), %d: 일(01-31)
-- %H:%i:%s 시:분:초
```

### 예: 2023년에 가입한 회원
```sql
SELECT * FROM users
WHERE YEAR(joined_at) = 2023;
```

## 조건 분기 (CASE WHEN)

```sql
SELECT name,
       CASE
           WHEN salary >= 10000 THEN '고액'
           WHEN salary >= 5000 THEN '중간'
           ELSE '낮음'
       END AS salary_level
FROM employees;
```

파이썬의 if-elif-else와 동일.

## 실행 순서 ⭐

SQL을 적는 순서와 **실제 처리 순서가 다름**.

```
적는 순서:    SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY → LIMIT
실제 순서:    FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT
```

→ WHERE에서는 SELECT의 별칭 못 씀. ORDER BY에서는 가능.

```sql
SELECT salary * 12 AS annual
FROM employees
WHERE annual > 50000      -- ❌ 에러 (WHERE는 SELECT 전에 처리)
ORDER BY annual DESC;     -- ✅ OK
```

## 코테 SQL 풀이 패턴

### 1. 기본 조회
```sql
SELECT 원하는_컬럼 FROM 테이블 WHERE 조건;
```

### 2. 정렬 + 상위 N개
```sql
SELECT * FROM 테이블 ORDER BY X DESC LIMIT N;
```

### 3. 조건부 조회
```sql
SELECT * FROM 테이블
WHERE 조건1 AND 조건2
ORDER BY 기준;
```

## 실전 팁

- **세미콜론**(`;`) 잊지 말기
- 문자열 비교는 **작은따옴표** `'IT'` (큰따옴표는 컬럼명용)
- NULL 비교는 `IS NULL` / `IS NOT NULL` (= NULL 안 됨)
- 컬럼명에 공백/예약어 있으면 백틱 (MySQL): `` `order` ``
