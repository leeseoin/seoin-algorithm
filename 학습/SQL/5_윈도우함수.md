# 윈도우 함수 (Window Functions)

## 한 줄 요약
**그룹별 집계를 하면서도 원본 행을 유지**하는 함수. GROUP BY는 행이 합쳐지지만, 윈도우 함수는 **각 행 옆에 그룹 통계를 같이 보여줌**.

## GROUP BY vs 윈도우 함수

### GROUP BY — 행이 줄어듦
```sql
SELECT dept_id, AVG(salary)
FROM employees
GROUP BY dept_id;
```
```
dept_id  AVG
1        5500
2        4800
```

### 윈도우 함수 — 행이 그대로 유지
```sql
SELECT name, dept_id, salary,
       AVG(salary) OVER (PARTITION BY dept_id) AS dept_avg
FROM employees;
```
```
name   dept_id  salary  dept_avg
Kim    1        5000    5500
Lee    1        6000    5500
Park   2        4800    4800
...
```

→ 각 사원의 **자기 부서 평균**을 옆에 같이 봄. 모든 행 유지.

## 기본 문법

```sql
함수() OVER (
    [PARTITION BY 그룹기준]
    [ORDER BY 정렬기준]
    [ROWS BETWEEN ... AND ...]
)
```

- `PARTITION BY`: 어떻게 그룹 나눌지 (없으면 전체가 하나의 윈도우)
- `ORDER BY`: 그룹 내 정렬 (RANK 등에 필요)
- `ROWS BETWEEN`: 범위 (이동 평균 등)

## 1. 순위 함수

### ROW_NUMBER() — 그냥 번호
```sql
SELECT name, salary,
       ROW_NUMBER() OVER (ORDER BY salary DESC) AS rn
FROM employees;
```
```
Kim    9000   1
Lee    8000   2
Park   8000   3   ← 동점이어도 다른 번호
Choi   7000   4
```

### RANK() — 동점 같은 순위, 다음은 건너뜀
```
Kim    9000   1
Lee    8000   2
Park   8000   2
Choi   7000   4   ← 3 건너뜀
```

### DENSE_RANK() — 동점 같은 순위, 다음은 안 건너뜀
```
Kim    9000   1
Lee    8000   2
Park   8000   2
Choi   7000   3   ← 3 (안 건너뜀)
```

### NTILE(n) — n 등분
```sql
SELECT name, salary, NTILE(4) OVER (ORDER BY salary) AS quartile
FROM employees;
```
4분위로 나눔.

## 2. 그룹별 순위 (PARTITION BY)

**부서별 연봉 순위**:
```sql
SELECT name, dept_id, salary,
       RANK() OVER (PARTITION BY dept_id ORDER BY salary DESC) AS dept_rank
FROM employees;
```
```
name   dept_id  salary  dept_rank
Kim    1        9000    1
Lee    1        7000    2
Park   1        5000    3
Choi   2        8000    1
Han    2        6000    2
...
```

→ "**각 그룹의 N등 찾기**" 문제의 핵심.

## 3. 집계 함수 + 윈도우

GROUP BY로 했던 함수들이 윈도우 함수로도 가능.

```sql
SELECT name, dept_id, salary,
       AVG(salary) OVER (PARTITION BY dept_id) AS dept_avg,
       MAX(salary) OVER (PARTITION BY dept_id) AS dept_max,
       COUNT(*) OVER (PARTITION BY dept_id) AS dept_size,
       SUM(salary) OVER (PARTITION BY dept_id) AS dept_total
FROM employees;
```

원본 행 유지하면서 각종 통계를 같이 봄.

## 4. LAG / LEAD — 이전/다음 행 값

```sql
SELECT date, sales,
       LAG(sales, 1) OVER (ORDER BY date) AS prev_day,
       LEAD(sales, 1) OVER (ORDER BY date) AS next_day,
       sales - LAG(sales, 1) OVER (ORDER BY date) AS change
FROM daily_sales;
```

```
date        sales   prev_day   next_day   change
2024-01-01  100     NULL       150        NULL
2024-01-02  150     100        130        50
2024-01-03  130     150        180       -20
```

→ "전일 대비 증감", "이전 값과 비교" 같은 시계열 분석에 강력.

## 5. 누적 (Running Total)

```sql
SELECT date, sales,
       SUM(sales) OVER (ORDER BY date) AS running_total
FROM daily_sales;
```

```
date        sales   running_total
2024-01-01  100     100
2024-01-02  150     250
2024-01-03  130     380
```

`ORDER BY`만 두고 PARTITION BY 없으면 전체 누적. 그룹별 누적이면 PARTITION BY 추가.

## 6. 이동 평균 (Moving Average)

`ROWS BETWEEN ... AND ...` 사용.

```sql
SELECT date, sales,
       AVG(sales) OVER (
           ORDER BY date
           ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
       ) AS moving_avg_3
FROM daily_sales;
```

→ 최근 3일 평균.

## 7. FIRST_VALUE / LAST_VALUE

```sql
-- 부서별 가장 연봉 높은 사람의 이름을 모든 행에 같이
SELECT name, dept_id, salary,
       FIRST_VALUE(name) OVER (
           PARTITION BY dept_id
           ORDER BY salary DESC
       ) AS top_earner
FROM employees;
```

## 자주 쓰는 패턴

### 1. 그룹별 N등
"각 부서에서 연봉 1등인 사람":
```sql
WITH ranked AS (
    SELECT *, RANK() OVER (PARTITION BY dept_id ORDER BY salary DESC) AS rk
    FROM employees
)
SELECT * FROM ranked WHERE rk = 1;
```

### 2. 그룹별 N등 안의 사람 (Top-N)
```sql
WITH ranked AS (
    SELECT *, ROW_NUMBER() OVER (PARTITION BY dept_id ORDER BY salary DESC) AS rn
    FROM employees
)
SELECT * FROM ranked WHERE rn <= 3;
```

### 3. 동일 그룹 내 비교
```sql
SELECT name, salary,
       AVG(salary) OVER (PARTITION BY dept_id) AS dept_avg,
       salary - AVG(salary) OVER (PARTITION BY dept_id) AS diff_from_avg
FROM employees;
```

### 4. 누적 비율
```sql
SELECT date, sales,
       sales / SUM(sales) OVER () * 100 AS pct
FROM daily_sales;
```

### 5. 변화량
```sql
SELECT date, value,
       value - LAG(value) OVER (ORDER BY date) AS change
FROM history;
```

## 실행 순서 — 윈도우 함수는 거의 마지막

```
FROM → WHERE → GROUP BY → HAVING → SELECT(윈도우) → ORDER BY → LIMIT
```

→ 윈도우 함수 결과는 **WHERE/HAVING에서 못 씀**. 쓰려면 CTE/서브쿼리로 한 번 감싸기.

```sql
-- ❌ 안 됨
SELECT * FROM employees
WHERE RANK() OVER (...) = 1;

-- ✅ CTE로
WITH ranked AS (
    SELECT *, RANK() OVER (...) AS rk FROM employees
)
SELECT * FROM ranked WHERE rk = 1;
```

## 윈도우 함수 vs 서브쿼리

같은 결과 여러 방법.

```sql
-- 부서별 1등 (서브쿼리)
SELECT * FROM employees e
WHERE salary = (SELECT MAX(salary) FROM employees WHERE dept_id = e.dept_id);

-- 윈도우 함수
WITH ranked AS (
    SELECT *, RANK() OVER (PARTITION BY dept_id ORDER BY salary DESC) AS rk
    FROM employees
)
SELECT * FROM ranked WHERE rk = 1;
```

윈도우 함수가 보통 **빠르고 가독성 좋음**.

## 신호 — 윈도우 함수 문제

- "**그룹별 N등**", "각 카테고리에서 가장 ~"
- 행을 유지하면서 그룹 통계 같이 보고 싶을 때
- "전일 대비", "직전 값과 비교"
- 누적 합, 이동 평균
- "전체 대비 비율"

## 실전 팁

- ROW_NUMBER vs RANK vs DENSE_RANK 차이 명확히
- "그룹별 1등" → ROW_NUMBER (동점 무관 1명) / RANK (동점 다 1등)
- 결과를 WHERE에 쓰려면 CTE 감싸기
- MySQL 8.0+ 에서 지원 (이전 버전은 안 됨)

## 대표 문제 유형 (프로그래머스 SQL 고득점 키트)

- 입양 시각 구하기, 식품분류별 가장 비싼 식품
- 우유와 요거트가 담긴 장바구니
- 카테고리별 도서 판매량 집계 (윈도우 함수 + 조인)
