# 해시테이블 (Hash Table)

## 한 줄 요약
**키 → 값**을 빠르게 찾을 수 있는 자료구조. 파이썬에선 `dict`(키-값)와 `set`(키만)이 곧 해시테이블. **검색/삽입/삭제가 평균 O(1)**.

## 어떻게 O(1)이 가능한가 (개념)

키를 **해시 함수**에 넣으면 정수 인덱스가 나옴. 그 인덱스 위치에 값 저장.
```
key "apple" → hash() → 17 → 배열[17]에 저장
key "banana" → hash() → 4 → 배열[4]에 저장
```

다음에 `"apple"`을 찾을 때도 같은 해시값이 나오니 배열[17]을 바로 봄. 검색하지 않고 **계산으로 위치를 찾음**.

⚠️ **충돌**(다른 키의 해시값이 같을 때) 처리는 내부 구현이 알아서 함. 파이썬 사용자는 신경 X.

## 파이썬에선 dict / set이 곧 해시테이블

### dict — 키-값 매핑
```python
d = {"apple": 100, "banana": 50}

d["cherry"] = 80          # 삽입, O(1)
print(d["apple"])          # 검색, O(1)
print(d.get("grape", 0))   # 없으면 기본값
del d["apple"]             # 삭제, O(1)
"banana" in d              # 키 존재 확인, O(1)
len(d)
```

### set — 키만 (중복 없는 모음)
```python
s = {1, 2, 3}

s.add(4)                  # O(1)
s.remove(2)               # O(1)
3 in s                    # O(1)

# 집합 연산
{1,2,3} & {2,3,4}         # {2,3}    교집합
{1,2,3} | {2,3,4}         # {1,2,3,4} 합집합
{1,2,3} - {2,3,4}         # {1}      차집합
{1,2,3} ^ {2,3,4}         # {1,4}    대칭차
```

### list와의 비교

| 연산 | `list` | `set` / `dict` |
|---|---|---|
| `x in container` | **O(n)** | **O(1)** ⭐ |
| 삽입 | append O(1) | add O(1) |
| 삭제 (값으로) | O(n) | O(1) |
| 인덱스 접근 | O(1) | 없음 |

코테에서 `x in list`가 자주 등장 → **시간 초과의 주범**. `set` 사용 빈도 ⭐⭐⭐.

## 시간복잡도 주의

| 연산 | 평균 | 최악 |
|---|---|---|
| 검색/삽입/삭제 | O(1) | O(n) (충돌 다발 시) |

실전에선 거의 항상 O(1). 최악 케이스는 일부러 만들지 않는 한 안 만남.

## 언제 쓰나 — 대표 패턴

### 1. 카운팅 / 빈도수
```python
from collections import Counter

cnt = Counter("aabbbc")          # {'a':2, 'b':3, 'c':1}
cnt.most_common(1)               # [('b', 3)]
```

### 2. 중복 제거
```python
unique = list(set(arr))
len(set(arr)) == len(arr)        # 중복 있는지
```

### 3. "이미 봤나?" 검사 (Visited Set)
BFS/DFS의 핵심.
```python
visited = set()
visited.add(node)
if next_node not in visited: ...
```

### 4. Two Sum 류 (보완 검색)
```python
def two_sum(arr, target):
    seen = {}
    for i, x in enumerate(arr):
        if target - x in seen:
            return [seen[target - x], i]
        seen[x] = i
```
O(n²) → O(n)으로 줄이는 핵심 기법.

### 5. 그룹화 (defaultdict)
```python
from collections import defaultdict

groups = defaultdict(list)
for word in words:
    key = "".join(sorted(word))   # 알파벳순 정렬을 키로
    groups[key].append(word)      # 같은 키끼리 묶임
```

### 6. 매핑 변환
```python
# 단어→번호
word_to_id = {w: i for i, w in enumerate(words)}
# 번호→단어
id_to_word = {i: w for i, w in enumerate(words)}
```

## 키로 쓸 수 있는 것 / 없는 것

해시 가능한(hashable) 객체만 키로 쓸 수 있음.

| 가능 | 불가능 |
|---|---|
| `int`, `str`, `float`, `bool` | `list` (가변) |
| `tuple` (안에 hashable만) | `dict` |
| `frozenset` | `set` |

```python
d = {(1, 2): "ok"}        # 튜플 OK
d = {[1, 2]: "fail"}      # ❌ TypeError
```

좌표 같은 거 키로 쓰려면 튜플 `(x, y)`.

## 신호 — 이 문제는 해시일 수 있다

- "**중복**", "유일한", "고유"
- "빈도", "가장 많은", "k번째"
- "이미 본 적 있나" (방문 체크)
- "쌍을 찾기" (Two Sum 류)
- "그룹화", "분류"
- 검색이 **반복적으로** 일어남 → list로는 O(n²) 가능성

## defaultdict 활용

`dict`를 쓸 때 키가 없으면 KeyError. `defaultdict`는 자동으로 기본값 생성.
```python
from collections import defaultdict

d = defaultdict(int)         # 기본 0
d["a"] += 1                  # KeyError 안 남

d = defaultdict(list)        # 기본 []
d["group1"].append(item)

d = defaultdict(set)         # 기본 set()
```

`d.setdefault(key, default)`도 비슷한 역할. 한 번만 쓸 거면 이게 더 명시적.

## 실전 팁

- "검색이 잦다" → 일단 `set` / `dict` 떠올리기
- `Counter`는 빈도수 문제의 만능키
- 좌표는 튜플 `(x, y)`로 키 만들기
- `defaultdict(list)`로 그룹화는 진짜 자주 나옴
