# 링크드리스트 (Linked List)

## 한 줄 요약
각 노드가 **다음 노드를 가리키는 화살표**로 연결된 자료구조. 보물찾기 단서 종이가 다음 단서 위치를 알려주는 거랑 같음.

```
[1|→]──→[2|→]──→[3|→]──→[4|None]
head                          tail
```

## list랑 뭐가 다름?

| | 배열(`list`) | 링크드리스트 |
|---|---|---|
| 메모리 | 연속된 공간 | 흩어져 있음, 포인터로 연결 |
| 인덱스 접근 | O(1) ⭐ | O(n) — 처음부터 따라가야 |
| 중간 삽입/삭제 | O(n) — 밀어야 함 | O(1) — 화살표만 바꾸면 |
| 끝 추가 | 보통 O(1) | O(1) (tail 알면) |

→ "**중간 삽입/삭제가 잦으면** 링크드리스트, **인덱스 접근이 잦으면** 배열".

## 파이썬에선 거의 안 짬

파이썬 코테에서 **직접 링크드리스트를 구현할 일은 드물어요**. `list`나 `deque`로 거의 다 해결됨. 그래도 개념과 구현은 면접에서 묻고, **두 포인터** 같은 패턴의 기반이라 알아둬야 함.

## 구현 (학습용, 한 번이면 충분)

### 단순 연결 (singly linked list)
```python
class Node:
    def __init__(self, value, next=None):
        self.value = value
        self.next = next

class LinkedList:
    def __init__(self):
        self.head = None

    def append(self, value):
        if not self.head:
            self.head = Node(value)
            return
        cur = self.head
        while cur.next:
            cur = cur.next
        cur.next = Node(value)

    def remove(self, value):
        if not self.head:
            return
        if self.head.value == value:
            self.head = self.head.next
            return
        cur = self.head
        while cur.next and cur.next.value != value:
            cur = cur.next
        if cur.next:
            cur.next = cur.next.next   # 화살표 건너뛰기

    def to_list(self):
        result, cur = [], self.head
        while cur:
            result.append(cur.value)
            cur = cur.next
        return result
```

### 이중 연결 (doubly linked list)
각 노드에 `prev`도 있음. 양방향 이동 가능.
- 파이썬 `collections.deque`가 내부적으로 doubly linked list로 구현됨.

## 시간복잡도

| 연산 | 복잡도 |
|---|---|
| 맨 앞 삽입/삭제 | O(1) |
| 맨 뒤 삽입 (tail 모르면) | O(n) |
| 인덱스 i 접근 | O(n) |
| 검색 | O(n) |
| 중간 삽입/삭제 (위치 알 때) | O(1) |

## 면접/응용에서 자주 나오는 문제

### 1. 사이클 감지 (Floyd's Tortoise and Hare)
**두 포인터** — 느린 거(1칸씩) + 빠른 거(2칸씩). 만나면 사이클 있음.
```python
def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return True
    return False
```

### 2. 뒤집기 (Reverse Linked List)
화살표 방향 뒤집기. `prev`, `cur`, `next` 세 변수.
```python
def reverse(head):
    prev = None
    cur = head
    while cur:
        nxt = cur.next
        cur.next = prev
        prev = cur
        cur = nxt
    return prev
```

### 3. 두 정렬된 리스트 병합
머지 소트의 핵심 로직.

### 4. 중간 노드 찾기
두 포인터 (느림+빠름). 빠른 게 끝나면 느린 게 중간.

## 두 포인터 패턴

링크드리스트뿐 아니라 **배열에서도 자주 쓰는 강력한 기법**.

```python
# 정렬된 배열에서 합이 target인 두 수 찾기
def two_sum(arr, target):
    left, right = 0, len(arr) - 1
    while left < right:
        s = arr[left] + arr[right]
        if s == target: return (left, right)
        elif s < target: left += 1
        else: right -= 1
    return None
```

링크드리스트의 사이클 감지도 두 포인터의 응용.

## 실전 팁

- 파이썬 코테에서 직접 LinkedList 클래스 만들 일은 드물다 (`list`/`deque` 활용)
- 하지만 **두 포인터 사고법**은 배열/문자열에서도 자주 씀 → 익혀둘 가치 충분
- `head`, `tail`, `prev`, `cur`, `next` 변수명 패턴 익숙해지기
