# 스택 (Stack)

## 한 줄 요약
**나중에 들어온 게 먼저 나가는** (LIFO, Last In First Out) 자료구조. 책상에 책을 쌓아 올렸다가 위에서부터 빼는 거랑 같음.

## 동작
```
push(1)  →  [1]
push(2)  →  [1, 2]
push(3)  →  [1, 2, 3]
pop()    →  3 반환, [1, 2]
pop()    →  2 반환, [1]
peek()   →  1 (제거 X, 보기만)
```

가장 최근에 push한 것만 접근 가능. 중간 원소는 못 봄.

## 파이썬 구현 — `list`로 끝

파이썬은 따로 스택 클래스가 없고 **`list`가 곧 스택**.

```python
stack = []

stack.append(1)        # push → O(1)
stack.append(2)
stack.append(3)
print(stack.pop())     # 3 (가장 위)
print(stack[-1])       # peek (제거 안 함)
print(len(stack))      # 크기
print(not stack)       # 비었는지 (True/False)
```

`list.append`와 `list.pop()` 둘 다 **O(1)** 이라 이게 곧 표준.

## 직접 구현해본다면 (학습용, 1번만)

```python
class Stack:
    def __init__(self):
        self.data = []

    def push(self, x):
        self.data.append(x)

    def pop(self):
        if not self.data:
            return None
        return self.data.pop()

    def peek(self):
        return self.data[-1] if self.data else None

    def is_empty(self):
        return len(self.data) == 0
```

실전에선 `list` 그대로 씀. 위 코드는 한 번 손으로 짜보고 잊어도 됨.

## 시간복잡도

| 연산 | 복잡도 |
|---|---|
| push | O(1) |
| pop | O(1) |
| peek | O(1) |
| 검색 (`x in stack`) | O(n) |

## 언제 쓰나 — 대표 패턴

### 1. 괄호/짝 검사
"여는 괄호는 push, 닫는 괄호는 가장 최근 여는 괄호와 매칭"
```python
def is_valid(s):
    stack = []
    pairs = {')': '(', ']': '[', '}': '{'}
    for ch in s:
        if ch in '([{':
            stack.append(ch)
        else:
            if not stack or stack.pop() != pairs[ch]:
                return False
    return not stack
```

### 2. 되돌리기 / 이전 상태
브라우저 뒤로가기, Ctrl+Z, 함수 호출 (콜 스택)
- 가장 최근 작업을 되돌려야 함 → 스택

### 3. DFS (깊이 우선 탐색)
재귀 호출 자체가 스택 (콜 스택). 명시적 스택으로도 짤 수 있음.
```python
stack = [start]
while stack:
    node = stack.pop()
    # 처리
    for next_node in neighbors(node):
        stack.append(next_node)
```

### 4. 수식 계산 (후위 표기법)
연산자 우선순위, 중위→후위 변환

### 5. 다음 큰 값 (Next Greater Element)
"오른쪽에서 자기보다 큰 첫 값" 류의 문제. 단조 스택(monotonic stack) 패턴.

## 신호 — 이 문제는 스택일 수 있다

- "최근의 ~", "마지막에 들어온 ~"
- 괄호, 짝, 매칭
- 되돌리기, undo
- 깊이 탐색, 재귀
- "왼쪽에서 가장 가까운 ~" / "오른쪽에서 다음으로 큰 ~"

## 실전 팁

- 파이썬은 그냥 `list`. 새 클래스 만들지 말 것.
- 비었는지 검사는 `if not stack:` 이 가장 파이썬다움.
- `stack.pop(0)`는 큐 동작인데 O(n)이라 **절대 쓰지 말 것**. 진짜 큐가 필요하면 `deque`.
