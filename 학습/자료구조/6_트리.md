# 트리 (Tree)

## 한 줄 요약
**사이클이 없는, 위에서 아래로 뻗어가는 그래프**. 가족 족보, 회사 조직도, 폴더 구조 같은 거.

```
        1            ← 루트(root)
       / \
      2   3
     /\   |
    4  5  6          ← 리프(leaf)
```

## 용어 정리

- **루트(root)** = 맨 위 노드 (시작점)
- **리프(leaf)** = 자식이 없는 노드 (끝)
- **부모/자식(parent/child)** = 위/아래 관계
- **형제(sibling)** = 같은 부모 가진 노드들
- **깊이(depth)** = 루트에서 떨어진 거리 (루트는 0)
- **높이(height)** = 가장 깊은 리프까지의 거리
- **서브트리(subtree)** = 어떤 노드를 루트로 보는 부분 트리

## 트리의 성질

- 노드가 N개면 간선은 **N-1개** (정확히)
- **사이클 없음** — 모든 두 노드 사이 경로가 유일
- 어떤 노드에서 출발해도 모두 도달 가능 (연결됨)
- → "**사이클 없고 연결된 그래프**"가 트리의 정의

## 이진 트리 (Binary Tree)

각 노드의 자식이 **최대 2개** (왼쪽/오른쪽). 가장 자주 다루는 형태.

```python
class Node:
    def __init__(self, value):
        self.value = value
        self.left = None
        self.right = None
```

### 이진 탐색 트리 (BST)
이진 트리 + 규칙: **왼쪽 < 부모 < 오른쪽**.
- 검색 O(log n) (균형 잡혔을 때)
- 파이썬 코테에선 직접 짤 일 거의 없음 (`set`/`dict`로 대체됨)

## 트리 순회 (Tree Traversal)

### DFS 기반 — 전위/중위/후위
```python
def preorder(node):    # 부모 → 왼 → 오
    if not node: return
    print(node.value)
    preorder(node.left)
    preorder(node.right)

def inorder(node):     # 왼 → 부모 → 오 (BST면 정렬된 순서!)
    if not node: return
    inorder(node.left)
    print(node.value)
    inorder(node.right)

def postorder(node):   # 왼 → 오 → 부모
    if not node: return
    postorder(node.left)
    postorder(node.right)
    print(node.value)
```

### BFS 기반 — 레벨 순회
같은 깊이의 노드를 함께 방문.
```python
from collections import deque

def level_order(root):
    if not root: return
    q = deque([root])
    while q:
        node = q.popleft()
        print(node.value)
        if node.left: q.append(node.left)
        if node.right: q.append(node.right)
```

## 일반 트리 표현 (자식 N개)

이진 트리가 아니면 자식을 리스트로.
```python
class Node:
    def __init__(self, value):
        self.value = value
        self.children = []
```

또는 **부모 배열**: `parent[i] = i의 부모 번호`.

또는 **그래프처럼**: `defaultdict(list)`로 인접 리스트 + 루트만 알면 됨.

## 자주 등장하는 트리 문제 패턴

### 1. 트리의 높이 / 깊이
재귀로 자식의 max + 1.
```python
def height(node):
    if not node: return -1
    return 1 + max(height(node.left), height(node.right))
```

### 2. 두 노드의 공통 조상 (LCA)
- 부모 배열 + 깊이 맞추기
- 또는 재귀로 위로 올라가며 비교

### 3. 트리의 지름 (가장 먼 두 노드 거리)
- 임의 노드에서 가장 먼 노드 A → A에서 가장 먼 노드 B → A-B가 지름
- DFS 두 번

### 4. 서브트리 합 / 카운트
재귀로 자식 결과를 부모에서 합치기.

### 5. 이진 트리 직렬화 / 복원
순회 결과로 트리 재구성

## 힙 (Heap) — 트리 기반 자료구조

**완전 이진 트리** + 규칙: **부모 ≤ 자식 (최소 힙)** 또는 **부모 ≥ 자식 (최대 힙)**.
- 루트에 항상 최소(최대)값
- 삽입/삭제 O(log n)
- **우선순위 큐**의 구현체

파이썬에선 `heapq` 모듈로 사용 (직접 트리 안 짜도 됨).
```python
import heapq
heap = []
heapq.heappush(heap, 3)
heapq.heappop(heap)        # 항상 최솟값
```

## 실전 팁

- 트리 문제는 **재귀가 자연스러움** — 자식 결과를 부모에서 합치는 패턴
- 깊이가 깊으면 재귀 한도 주의: `sys.setrecursionlimit(10**6)`
- BFS로 풀 수 있는 트리 문제는 보통 DFS로도 가능 (역도 비슷)
- 이진 트리 노드 직접 만들기보단 **그래프(인접 리스트)** 로 다루는 게 편할 때 많음
