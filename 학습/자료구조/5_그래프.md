# 그래프 (Graph)

## 한 줄 요약
**점(노드)들이 선(엣지)으로 연결된** 자료구조. 사람 친구 관계, 도시 도로망, 웹사이트 링크 같은 걸 모델링.

```
    1───2
    │   │
    3───4
    │
    5
```

노드: {1,2,3,4,5}, 엣지: {(1,2), (1,3), (2,4), (3,4), (3,5)}

## 용어 정리

- **정점(vertex/node)** = 점
- **간선(edge)** = 선
- **방향 그래프** = 화살표 있음 (A→B는 가능, B→A는 별개)
- **무방향 그래프** = 화살표 없음 (A↔B는 양쪽 다 가능)
- **가중치 그래프** = 간선마다 비용/거리 있음 (도시 간 거리)
- **사이클** = 출발 노드로 돌아오는 경로 있음
- **DAG** = 방향 + 사이클 없음 (Directed Acyclic Graph) — 위상 정렬의 대상

## 표현 방법 2가지

### 1. 인접 리스트 (가장 자주 씀)
각 노드별로 "이 노드와 연결된 이웃 리스트"를 저장.
```python
from collections import defaultdict

graph = defaultdict(list)
edges = [(1,2), (1,3), (2,4), (3,4), (3,5)]
for a, b in edges:
    graph[a].append(b)
    graph[b].append(a)        # 무방향이면 양쪽 다

# graph[1] → [2, 3]
# graph[3] → [1, 4, 5]
```

**가중치 있으면**:
```python
graph = defaultdict(list)
for a, b, w in weighted_edges:
    graph[a].append((b, w))   # (이웃, 가중치)
```

### 2. 인접 행렬
2D 배열. `matrix[a][b] = 1`이면 a와 b가 연결.
```python
n = 5
matrix = [[0]*n for _ in range(n)]
for a, b in edges:
    matrix[a][b] = 1
    matrix[b][a] = 1
```

| | 인접 리스트 | 인접 행렬 |
|---|---|---|
| 메모리 | O(V+E) | O(V²) |
| "a-b 연결됐나?" | O(이웃 수) | O(1) |
| 이웃 순회 | O(이웃 수) | O(V) |
| 언제 | **간선이 적을 때** (희소) | 간선이 많을 때 (밀집) |

코테는 보통 **인접 리스트**가 정답.

## 격자(grid)도 그래프

2D 격자에서 상하좌우 이동 = 각 칸이 노드, 인접 칸이 이웃인 그래프.
```python
dx = [-1, 1, 0, 0]
dy = [0, 0, -1, 1]

for d in range(4):
    nx, ny = x + dx[d], y + dy[d]
    if 0 <= nx < n and 0 <= ny < m:
        # 인접 칸 처리
```

이 패턴은 **BFS/DFS 격자 문제의 표준**.

## 그래프 위에서 할 수 있는 것

| 문제 | 알고리즘 |
|---|---|
| 모든 노드 방문 | DFS / BFS |
| **최단 거리** (간선 1) | BFS |
| **최단 거리** (가중치) | 다익스트라, 벨만-포드 |
| 모든 쌍 최단 거리 | 플로이드-워셜 |
| 사이클 있나? | DFS (방문 색칠) |
| 위상 정렬 (작업 순서) | 큐 + indegree |
| 연결 요소 개수 | DFS/BFS 반복 |
| 최소 비용으로 모두 연결 | MST (크루스칼/프림) |

## 핵심 — visited 처리

그래프 탐색의 90%는 **방문 표시**. 안 하면 무한 루프.
```python
visited = set()       # 또는 [False]*n
visited.add(start)

# 탐색 중
if next_node not in visited:
    visited.add(next_node)
    # 처리
```

## 신호 — 이 문제는 그래프일 수 있다

- "관계", "연결", "친구", "도로", "네트워크"
- "최단 거리", "최소 이동"
- "모두 도달 가능한지", "갇혀 있는지"
- 격자에서 이동 (상하좌우)
- "X에서 Y로 가는 방법"

## 실전 팁

- 인접 리스트는 `defaultdict(list)` 표준
- 무방향이면 **양쪽 다 추가** 잊지 말 것
- 격자 문제는 `dx`, `dy` 방향 배열 익숙해지기
- `visited`는 `set`이 깔끔 (중복 체크 O(1))
- 노드 번호가 1부터면 배열 크기 `n+1`
