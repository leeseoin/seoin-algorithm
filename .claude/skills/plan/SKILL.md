# 구현 계획 수립

$ARGUMENTS 에 대한 구현 계획을 수립한다. 인자가 없으면 현재 대화 맥락에서 작업 주제를 파악한다.

## 절차

1. **현재 상태 파악**
   - 관련 코드/설정 파일 읽기
   - `docs/TODO.md` 확인하여 기존 계획과 충돌 없는지 확인

2. **계획 수립**
   - Plan 모드로 진입하여 구현 계획 작성
   - 작업을 단계별로 분리 (각 단계는 독립적으로 테스트 가능해야 함)
   - 각 단계에 예상 변경 파일 목록 포함
   - 리스크/주의사항 명시

3. **계획 출력 형식**
   ```
   ## [주제] 구현 계획

   ### 배경
   - 왜 이 작업이 필요한지

   ### 단계별 계획
   1. **단계명** — 설명
      - 변경 파일: ...
      - 핵심 작업: ...

   ### 리스크/주의사항
   - ...

   ### 완료 기준
   - ...
   ```

4. **문서 작성**
   - 계획 내용을 `docs/` 디렉토리에 마크다운 문서로 저장
   - 파일명: 주제에 맞는 이름 (예: `docs/reports/HLS_성능분석_계획.md`)
   - 문서에 배경, 단계별 계획, 완료 기준 포함
   - `/update-docs todo`로 TODO.md에도 반영

5. **사용자 확인**
   - 계획을 보여주고 사용자 동의를 구한 후 작업 시작

## 주의사항
- 계획은 최대한 간결하게, 불필요한 단계 없이
- 기존 코드를 최대한 활용 (새 파일 최소화)
- 계획 문서는 항상 `docs/` 하위에 저장
