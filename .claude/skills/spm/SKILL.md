# 업무보고 일지 작성 (SPM)

$ARGUMENTS 날짜의 업무보고를 작성한다. 인자 없으면 오늘.

## 절차

1. 오늘 작업 내용 + `git log --oneline -10` 확인
2. `docs/reports/업무보고/업무보고_YYYYMMDD.md` 파일 작성
3. 사용자에게 보여주고 확인

## 형식

```
# 업무보고 - YYYY-MM-DD

## 업무내용
1) 큰 분류 1
2) 큰 분류 2

## 진행상황

### 1.1) 작업명
- 핵심 한 줄
- 결과/수치

### 2.1) 작업명
- 핵심 한 줄

## To do list
1) 다음 작업
2) 다음 작업
```

## 작성 원칙

- **짧게**: 각 작업 항목당 **1~3줄 bullet**. 구구절절 쓰지 않음.
- **결과 우선**: "~했음" 보다 "~완료 / 정상 동작 확인 / XX% 개선" 식으로
- **번호 매칭**: 업무내용 `1)` → 진행상황 `1.1, 1.2`
- **팀장님이 이해 가능한 수준** 용어 사용
- 파일 경로: `docs/reports/업무보고/업무보고_YYYYMMDD.md`
- 프로젝트 루트: `/Users/iseoin/SpringBoot_project/webrtc`
